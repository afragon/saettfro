Växthuset

Ett växthus fungerar ungefär som jordens atmosfär. Stora delar av solens energi strömmar igenom glasrutorna och värmer upp luften i växthuset. Glasrutorna hindrar sedan den uppvärmda luften från att strömma ut ur växthuset, detta gör att luften i huset blir mycket varmare än luften utanför. På det viset kan man odla olika typer av växter som annars inte klarar av det kallare klimatet genom att reglera luftfuktighet och temperatur som passar dem. Att underhålla ett växthus och odla växter som kräver en viss temperatur, fuktighet och/eller vattenmängd i odlingen kräver både tid och energi.

Tänk om det fanns ett sätt att låta allting regleras automatiskt?

Det är här ett automatiserat växthus kommer in. Ett automatiserat växthus består bland annat av olika sensorer som mäter de viktigaste faktorerna i en odling, exempelvis temperatur. Med hjälp av programmering går det sedan att göra så att en lucka i taket öppnas och stängs med hjälp av en motor baserat på temperaturen. Du kan även starta vattenpumpen om det skulle bli för torrt i växthuset.

Steg 4: Observera

Ni kan nu observera växthuset och se hur det fungerar. Prova att plantera lite olika frön och se om de börjar växa, exempelvis kryddor då resultat brukar ske inom några dagar. Viktigt att tänka på när ni planterar är att blöta jorden innan så den blir lätt fuktig. Detta gör att det räcker med mindre bevattning senare.



Efter några dagar kan ni ställa er frågan om växthuset fungerar som det ska eller om ni behöver justera några värden. Har saker börjat växa eller syns det inget? Om det inte växer något kan det vara bra att kolla upp vad växterna ni planterat behöver ha för miljö eller om dessa ens lämpar sig för ett växthus. Det kan vara så enkelt som att temperaturen behöver vara lägre eller att växten är känslig för övervattning.



Utökning av växthuset


Börja med att lägga till Wi-Fi modulen som är valfri och prova att skicka upp data till thingspeak. Om ni vill göra en egen hemsida som ni designar själva och visar information om växthuset på kan ni koppla er arduino som sköter växthuset till en Raspberry Pi t.ex. Med hjälp av den kan ni läsa data från arduinon och skapa en webserver med hjälp av t.ex. Python. Sedan kan ni skapa eran hemsida med hjälp av HTML, CSS och Javascript.

Ni skulle även kunna fundera på utökningar av själva växthuset, fler sensorer t.ex. Hade det varit intressant att mäta fuktigheten i jorden? Vad hade det gett för fördelar? Nackdelar? Ni kan även ställa er frågan om en lampa för att simulera solen kan vara nödvändigt om det är mörkare. Vad finns det för för och nackedelar med en lampa?

Ni kan också fråga er om det finns något bättre sätt att utforma växthuset på. Går det att vattna på ytterligare sätt? Kan man fånga upp spillvatten som missar krukorna vid takbevattning? Kan man mäta flera saker och justera andra saker för att ge växterna en ännu bättre miljö? Kanske en arm som gödslar en gång i veckan.

Var inte rädda för att prova andra saker med växthuset och förbättra det, det är lite poängen med det att ni ska komma på förbättringar. Tänk på att det finns inga gränser.

Basen

Först ska vi bygga basen. Basen kommer vara “golvet” till huset och träskivan kommer vara till hjälp när huset flyttas runt.




  1. Ta fram 3 st träreglar för kortsidan, 2 st träreglar för långsidan samt en träskiva. Placera långsidorna på träskivan enligt bilden nedan.

  2. Placera sedan tre kortsidor: en på vänstersida, en på högersida och en på mitten (för stabilitet). Sätt sedan dit plexiglaset ovanpå träreglarna, detta kommer att bli basen till växthuset.

Hörnen

Fortsätt sedan med hörnen.


  1. Mät och lägg ut fyra trä-reglar i hörnen.
  2. Skruva fast träskivan med botten-plexiglaset och trä-reglarna nerifrån och upp.

  3. Se till att skruvarna är tillräckligt långa för att skruva ihop materialet samt en marginal för att få det stabilt.


Du kan även fästa träskivan och reglarna med hjälp av spikar, trälim eller annat slitstarkt fästningsmedel som kan hålla ihop delarna.



Kanterna

Pelarna som ska hålla taket upp sätts på med hjälp av skruvar/spikar.


  1. Sätt dit ytterligare två pelare för stöd till taket.
  2. Sätt dit ytterligare två pelare för stöd till taket.
  3. Placera två reglar på vardera sida av pelarna. Skruva inte fast dem än. Det gör vi samtidigt när vi skruvar på husgavlarna i nästa steg.


Taket

Dags att bygga taket!


  1. Ta fyra nya träreglar så att det ser ut som husgavlar.
  2. Skruva fast husgavlarna, tillsammans med hörnen, uppifrån och ner.

  3. Skruva fast en lång träregel på toppen av huset. Sätt även dit kortsidorna för att göra klart husgavlarna.
  4. Sätt fast tre reglar för att hålla upp taket. Anledningen till att det är fyra reglar på ena sidan och tre reglar på andra sidan är för att fönstret ska få plats. Sätt även fast gångjärn till fönstret och någonting som kan hänga upp en liten vattenslang.


Om du borrar hål för skruvarna: var extra försiktig då träreglarna kan spricka.



Huset

Snart är huset klart!


  1. Skär ut plexiglas enligt husets mått som ska vara fyra väggar och flera mindre skivor som ska vara tak. Det är viktigt att glasen täcker hela huset så att det blir en bra isolering.

  2. Borra ett litet hål i plexiglaset så att vattenslangen kan gå igenom.

  3. Glöm inte sätta fast en eller två plexiglas med gångjärnen så att det blir takfönster. Använd gärna en vanlig petflaska för vattnet, samt en liten låda för elektroniken.


TEMPERATUR

En temperatursensor är en viktig del inom olika övervakningssystem. Med hjälp av dessa värden kommer vi kontrollera servot och fläkten. Vi kommer att använda temperatur- och fuktighetssensorn DHT-22 som kopplas enligt kopplingsschemat nedan.
Notera att en pinne inte används.


  1. Ground skall kopplas till den högra pinnen sett framifrån.
  2. Styrningspinen, som är den andra från vänster, skall kopplas till någon utav I/O portarna 1-13. Här har vi kopplat den i nummer 5.
  3. Den vänstra pinnen kopplas till 3.3 eller 5 V.
  4. Mellan Strömmen och styrningspinnen skall det sitta en 10K resistor enligt kopplingsschemat.

WiFi

Med hjälp av Wi-Fi modulen kommer man att kunna skicka data till en hemsida som heter thingspeak. Denna hemsida kan visa en graf över all data som skickas till den, tex temperaturen och fuktigheten i växthuset. För att det ska fungera behövs ett Wi-Fi nätverk som modulen kan koppla upp sig mot. För att uppkopplingen mellan Wi-Fi modulen och nätverket ska fungera behöver man veta nätverkets namn och lösenord. För att skicka upp data till thingspeak behöver man skapa ett konto och hämta något som kallas för en API-nyckel.


  1. Port 2 på arduinon skall kopplas till TX porten på wifi modulen. Detta syns på kopplingsschemat nedan.
  2. Ground från arduino skall kopplas via en kondensator innan den kopplas i ground på wifi modulen enligt schemat nedan. Ground skall även kopplas till den sista resistorn.
  3. 5V på arduinon kopplas med 2 dioder emellan innan den skall gå in i kondensatorn och kopplas till CH_PD och VCC porten på wifi modulen.
  4. I/O port 3 skall kopplas till slutet på resistorerna som är seriekopplade. En resistor efter detta kopplas en kabel till RX på wifimodulen enligt schemat.



SoftwareSerial men detta bibliotek stödjer inte en BAUD hastighet högre än 9600. Därför behöver vi ändra BAUD hastigheten på ESP8266. Detta görs enligt följande steg nedan.

  1. Sladden som sitter i pin 3 ska sitta i pin 0 (RX)

  2. Sladden som sitter i pin 2 ska sitta i pin 1 (TX)

  3. Ladda ett tomt program till Arduinon

  4. Öppna Serial Monitor

  5. Skicka in kommandot: ”AT+GMR”
    Detta för att kontrollera vilken ”AT firmware” som finns inprogrammerad på wi-fi modulen. Firmware är mjukvara som är inprogrammerad oftast på ROM eller flash minne.
  6. Skicka in kommandot ”AT+CIOBAUD=9600”

  7. För att koppla upp sig mot sitt Wi-Fi börjar man med att ladda upp ett tomt program till arduinon från dess IDE.

  8. För att ställa in rätt operationsläge börjar man med att skriva in AT. Detta kontrollerar att ESP8266 är okej. Denna ska ge “OK” om allting fungerar som det ska.

  9. Skriv in “AT+RST”. Detta startar om Wi-Fi modulen.

  10. Skriv in “AT+CWMODE=1”. Ställer in rätt operationsläge på modulen.

  11. För att nu koppla upp sig mot Wi-Fi börjar vi med att skriva “AT+CWLAP” som listar alla tillgängliga nätverk.

  12. Skriv sedan in “AT+CWJAP=”SSID”,”PASS” där SSID är namnet på ert nätverk och PASS är lösenordet till nätverket.

  13. Skriv till sist “AT+CWJAP?” för att visa vilket nätverk modulen är kopplad till, detta borde vara ditt nätverk.

  14. Klart! Sätt tillbaka sladdarna som de satt innan.


    15. Tips!
    AT-kommandon är instruktioner som används för att kontrollera ett modem (i vårt fall ESP8266).
    AT är en förkortning för ATtention. Alla kommandon börjar med ”AT”.

Fläkt

Fläkten kommer att användas för att cirkulera luften i växthuset vid hög temperatur och fuktighet.


  1. Fläkten kopplas enligt schemat nedan. Här är det precis som med pumpen extra viktigt att koppla rätt för att skydda Arduinon.
  2. Pin 8 på arduinon skall kopplas till mittenbenet på transistorn. Se bilden nedan. Denna skall kopplas med en 1K resistor emellan.
  3. Ground kopplas till det vänstra benet på transistorn. Sett med den platta sidan mot er.
  4. Det högra benet på transistorn skall gå till den negativa sidan på motorn.
  5. Strömmen kopplas till Vin porten på arduinon. Den kopplas sedan till fläkten.
  6. Transistorn är till för att skydda Arduinon från höga volttal.
  7. Strömmen kopplas till Vin porten på arduinon. Den kopplas sedan till fläkten.

Pump

Genom att ha en pump kan man pumpa vatten till växterna när de behöver vattnas.


  1. Här är det extra viktigt att koppla rätt för att skydda arduinon.
  2. Pin 7 på arduinon skall kopplas till mittenbenet på transistorn. Se bilden nedan. Denna skall kopplas med en 1K resistor emellan.
  3. Ground kopplas till det vänstra benet på transistorn.
  4. Det högra benet på transistorn kommer till slut att gå till den negativa sidan på motorn. Men den skall först kopplas med en diod emellan den och strömmen enligt kopplingsschemat nedan.
  5. Strömmen kopplas till Vin porten på arduinon. Den kopplas sedan genom samma spår som dioden innan den kopplas till plussidan på motorn.
  6. Diodern är till för att strömmen inte ska åka tillbaka in i arduinon och förstöra den.
  7. Transistorn är till för att skydda arduinon för höga volttal.

Servo

  3. Signal-kabeln är oftast gul eller orange och skall kopplas till en av I/O portarna 1-13 på Arduino.

Arduino

Vad är en Arduino och vad kan den göra?


Arduino är en bräda som kan läsa ingångar, exempelvis ljus på en sensor, ett finger på en knapp, eller ett Twitter-meddelande. Den skickar sedan en signal och aktiverar en motor, slår på en LED-lampa eller publicerar någonting på sociala medier. Du kan med en Arduino programmera vad den ska göra genom att skriva en uppsättning instruktioner i programvaran Arduino Software (IDE) med programmeringsspråket Arduino.

Varför ska vi använda Arduino?


Programvaran är enkel att använda för nybörjare och kan köras på Mac, Windows och Linux. Dessutom är Arduino-brädor relativt billiga jämfört med andra mikrokontrollsystem. Nedan är en bild på en Arduino Uno, brädan som vi har använt. Den har bland annat 14 digitala input/output-pinnar, USB-anslutning för att koppla direkt till datorn och en återställningsknapp - allt som behövs för att kunna koppla sensorer till ett miniväxthus.
Arduino Uno går att köpa via Arduinos hemsida.

För att minimera risken att flera processer körs samtidigt och överbelasta Arduinon använder vi oss av två metoder för att göra så att det endast är en process som körs åt gången. Dessa är pausProcess() och continueProcess().

Om Wifi modulen inte används behöver endast pausProcess() köras innan pumpen startas för att förhindra att pumpen och fläkten körs samtidigt. Om Wifi-modulen används behövs pausProcess även köras när Wifi-modulen körs.


/*
  1 = pump, 2 = ESP8266
*/
 void pausProcess(int typeOfDevice) {
  if (typeOfDevice == 1) {
    if (fanStatus) {
      stopFan();
      ontinueFan = true;
    }
  }
  if (typeOfDevice == 2) {
    if (fanStatus) {
      stopFan();
      continueFan = true;
    }
    if (pumpStatus) {
      stopPump();
      continuePump = true;
    }
  }
}

void continueProcess() {
  if (continueFan) {
    startFan();
    continueFan = false;
  }
  if (continuePump) {
    startPump();
    continuePump = false;
  }
}
